\capitulo{4}{Técnicas y herramientas}

En este capítulo se describen las metodologías, técnicas y herramientas tecnológicas utilizadas durante el desarrollo del trabajo, justificando los motivos principales de su selección y su aplicación dentro del proyecto. Asimismo, se presentan los entornos de desarrollo, sistemas de control de versiones y herramientas de documentación empleados durante la implementación del sistema.

\section{Metodologías de desarrollo}

Durante el desarrollo del proyecto se ha seguido un enfoque iterativo e incremental inspirado en metodologías ágiles. Este planteamiento ha permitido organizar el trabajo en diferentes fases de desarrollo, facilitando la planificación, evaluación y mejora continua de las funcionalidades implementadas.

\subsection{Scrum}

Scrum constituye un marco de trabajo ágil orientado a la gestión y desarrollo de proyectos software. Su funcionamiento se basa en iteraciones temporales denominadas \textit{sprints}, dentro de las cuales se desarrollan funcionalidades concretas del sistema.

En el contexto de este trabajo, se ha adoptado una adaptación simplificada de Scrum, organizando el desarrollo en ciclos de trabajo de corta duración y realizando revisiones periódicas del progreso alcanzado. Este enfoque ha permitido gestionar de forma estructurada la implementación de nuevas funcionalidades y la corrección de incidencias detectadas durante el desarrollo.

\subsection{Kanban}

Kanban es una metodología de gestión visual del trabajo orientada a optimizar el flujo de tareas y facilitar el seguimiento del estado del proyecto. Para ello, utiliza tableros organizados en columnas que representan las diferentes fases del desarrollo.

Durante el proyecto se ha empleado un sistema de organización basado en tableros Kanban mediante GitHub Projects, permitiendo gestionar tareas pendientes, funcionalidades en desarrollo y elementos completados de manera visual y estructurada.

\section{Tecnologías principales}

\subsection{Blender}

Blender constituye el entorno principal utilizado para el desarrollo del sistema. Se trata de un software libre y de código abierto orientado al modelado, animación y procesamiento de gráficos tridimensionales.

La elección de Blender se justifica por su arquitectura extensible y por la posibilidad de desarrollar funcionalidades personalizadas mediante \textit{add-ons}. Además, proporciona acceso programático a los elementos internos de la escena tridimensional, permitiendo monitorizar eventos y capturar información relacionada con el proceso de modelado.

\subsection{Python}

Python~\cite{pythonlang} ha sido utilizado como lenguaje principal de desarrollo debido a su integración nativa con Blender y a su amplio ecosistema de bibliotecas orientadas al análisis de datos y automatización de procesos.

La utilización de Python facilita la implementación de sistemas de captura de eventos, procesamiento de datos y generación de visualizaciones analíticas dentro del entorno tridimensional. Asimismo, su flexibilidad permite estructurar entornos portables mediante directorios internos de paquetes (\textit{site-packages}), asegurando la distribución del sistema sin alterar las dependencias globales del software anfitrión.

\subsection{Blender Python API (\texttt{bpy})}

La API oficial de Blender, accesible mediante el módulo \texttt{bpy}~\cite{blenderapi}, permite interactuar con la escena tridimensional, los objetos y las operaciones ejecutadas por el usuario.

A través de esta interfaz se accede a información relacionada con:
\begin{itemize}
\item Objetos y jerarquías de la escena activa.
\item Matrices de transformación espacial de coordenadas mundiales y locales.
\item Topología estructural de mallas mediante punteros nativos y submodelos \textit{BMesh}.
\item Modos de trabajo y conmutación de la interfaz gráfica.
\item Eventos y operadores ejecutados en el flujo de ejecución del usuario.
\end{itemize}

Asimismo, la API permite implementar sistemas de monitorización mediante la suscripción de funciones a los \textit{handlers} de la aplicación, utilizados para detectar automáticamente modificaciones analíticas durante las sesiones de modelado.

\subsection{Persistencia de datos en formato CSV}

El formato CSV (\textit{Comma-Separated Values}) se ha utilizado como mecanismo principal de almacenamiento de datos estructurados para el registro secuencial de interacciones.

La utilización de este formato se justifica por su simplicidad, compatibilidad técnica, nula sobrecarga en la memoria volátil y facilidad de integración con herramientas externas de análisis. Con el objetivo de optimizar la velocidad de acceso y la persistencia sin comprometer el rendimiento en tiempo real del visor 3D, el procesamiento de estos archivos tabulares se realiza mediante flujos lineales orientados a diccionarios empleando el módulo nativo \texttt{csv.DictReader}, evitando así la carga excesiva en memoria de bibliotecas de análisis masivo de datos.

\subsection{NumPy}

NumPy constituye una biblioteca especializada en computación numérica y procesamiento eficiente de estructuras matriciales en Python. Su utilización es crítica en este sistema para realizar operaciones vectoriales de alto rendimiento, el cálculo de las posiciones espaciales de los vértices evaluados y las transformaciones numéricas necesarias para proyectar los datos estadísticos sobre el espacio geométrico bidimensional y tridimensional.

\subsection{Matplotlib y Renderizado en Memoria por Canales}

Matplotlib~\cite{matplotlib} se ha empleado para la generación de las representaciones gráficas y visualizaciones de carácter inferencial y descriptivo.

A diferencia de los entornos de ejecución tradicionales de análisis científico, donde los gráficos se despliegan en ventanas independientes del sistema operativo a través de interfaces de usuario bloqueantes, en esta solución se implementa una técnica de desacoplamiento basada en el motor de renderizado abstracto de la biblioteca (\texttt{FigureCanvasAgg}). Esto permite generar los diagramas estadísticos y las matrices de correlación directamente en un espacio de memoria volátil como matrices de píxeles (\textit{raw buffers}), las cuales son inyectadas dinámicamente como texturas en tiempo de ejecución sobre mallas poligonales procedurales en el visor 3D de Blender.

\section{Herramientas de desarrollo}

\subsection{Visual Studio Code}

Visual Studio Code ha sido utilizado como entorno de desarrollo principal durante la implementación del proyecto. Este editor proporciona herramientas avanzadas de depuración de hilos, autocompletado sintáctico y gestión de extensiones, facilitando el desarrollo de código en Python y la integración con sistemas de control de versiones.

\subsection{Git y GitHub}

Git ha sido utilizado como sistema de control de versiones para gestionar el historial de cambios realizado durante el desarrollo del proyecto. Complementariamente, GitHub se ha utilizado como plataforma de alojamiento remoto del repositorio, permitiendo almacenar versiones del código, gestionar ramas de desarrollo y mantener un seguimiento organizado de la evolución del sistema.

\subsection{GitHub Projects}

GitHub Projects se ha empleado como herramienta de organización y planificación de tareas mediante tableros Kanban. Su utilización ha permitido estructurar el flujo de trabajo y mantener un seguimiento visual del estado de desarrollo de cada funcionalidad implementada.

\subsection{Overleaf y entornos locales de LaTeX}

La memoria del proyecto ha sido desarrollada utilizando \LaTeX{}, un sistema de composición tipográfica ampliamente utilizado en documentación científica y técnica. Inicialmente, se empleó la plataforma web Overleaf para la maquetación y edición colaborativa del documento. Posteriormente, debido al incremento en el volumen de imágenes y a la complejidad técnica de la compilación de bibliografías complejas, el flujo de trabajo principal se trasladó a un entorno local de compilación tipográfica, optimizando los tiempos de generación del documento final.
