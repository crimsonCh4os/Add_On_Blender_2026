\apendice{Especificación de Requisitos}

\section{Introducción}

En este apartado se presenta la especificación de requisitos del sistema desarrollado, compuesto por dos addons independientes pero complementarios para Blender:

\begin{itemize}
    \item Un addon de recopilación de datos, encargado de registrar la actividad del usuario durante el proceso de modelado 3D.
    \item Un addon de análisis y visualización, encargado de procesar los datos registrados y generar métricas y representaciones visuales.
\end{itemize}

En primer lugar, se describen los casos de uso más representativos del sistema. Posteriormente, se recogen los objetivos generales, los diagramas que modelan el comportamiento y la arquitectura del sistema, y finalmente el catálogo completo de requisitos funcionales, no funcionales, de interfaz y legales.

\section{Objetivos generales}

A partir de los casos de uso descritos, se establecen los objetivos generales que debe satisfacer el sistema:

\begin{itemize}
    \item Registrar de manera precisa los eventos relevantes producidos durante el modelado 3D.
    
    \item Permitir el análisis temporal, espacial y estructural de la actividad del usuario.
    
    \item Generar visualizaciones gráficas y tridimensionales de las métricas obtenidas.
    
    \item Garantizar la persistencia e integridad de los datos entre sesiones.
    
    \item Proporcionar una interfaz clara y accesible para ambos addons.
    
    \item Mantener una arquitectura modular y desacoplada.
    
    \item Cumplir con la normativa legal y ética relacionada con la protección de datos y el uso de software libre.
\end{itemize}

\section{Catálogo de requisitos}

\subsection{Requisitos funcionales (RF)}

\begin{itemize}
    \item RF-01: Registrar la creación de objetos.
    \item RF-02: Registrar modificaciones geométricas.
    \item RF-03: Detectar cambios de modo de edición.
    \item RF-04: Registrar eliminación de objetos.
    \item RF-05: Visualizar métricas de modelado.
    \item RF-06: Exportar datos para análisis externo.
    \item RF-07: Filtrar métricas por objeto, evento o tiempo.
    \item RF-08: Notificar errores de captura o procesamiento.
    \item RF-09: Activar y desactivar la captura desde la interfaz.
    \item RF-10: Registrar marcas temporales precisas.
    \item RF-11: Generar visualizaciones tridimensionales.
    \item RF-12: Integrar representaciones visuales dentro de Blender.
    \item RF-13: Generar gráficos estadísticos 2D.
\end{itemize}

\subsection{Requisitos no funcionales (RNF)}

\begin{itemize}
    \item RNF-01: Eficiencia en la captura de eventos.
    \item RNF-02: Soporte para múltiples objetos y escenas complejas.
    \item RNF-03: Persistencia de datos ante cierres inesperados.
    \item RNF-04: Interfaz clara e intuitiva.
    \item RNF-05: Código modular y mantenible.
    \item RNF-06: Bajo impacto en el rendimiento de Blender.
\end{itemize}

\subsection{Requisitos de interfaz de usuario (RIU)}

\begin{itemize}
    \item RIU-01: Panel de control de captura.
    \item RIU-02: Panel de análisis y métricas.
    \item RIU-03: Configuración de exportación de datos.
    \item RIU-04: Alertas y mensajes de error.
    \item RIU-05: Controles para visualizaciones 2D y 3D.
\end{itemize}

\subsection{Requisitos legales y éticos (RLE)}

\begin{itemize}
    \item RLE-01: Cumplimiento de la normativa de protección de datos.
    \item RLE-02: Consentimiento explícito del usuario para la recopilación de información.
    \item RLE-03: Uso de software libre respetando las licencias aplicables.
\end{itemize}
