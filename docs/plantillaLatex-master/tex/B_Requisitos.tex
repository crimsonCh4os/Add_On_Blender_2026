\apendice{Especificación de Requisitos}

En este apartado se presentan los casos de uso representativos de los dos addons desarrollados en el proyecto:  
el addon de recopilación de datos y el addon de análisis de datos y métricas. Cada caso de uso incluye información detallada de precondiciones, acciones, postcondiciones y posibles excepciones.

\section{Objetivos generales}

\begin{itemize}
    \item Registrar de manera precisa los eventos relevantes durante el modelado 3D.
    \item Permitir análisis estadístico y visualización de los datos recopilados.
    \item Garantizar persistencia, integridad y trazabilidad de los datos entre sesiones.
    \item Proporcionar una interfaz clara y segura para la interacción con los addons.
    \item Cumplir con la normativa legal y ética de protección de datos.
\end{itemize}

\section{Catálogo de requisitos}

\subsection{Requisitos funcionales (RF)}
\begin{itemize}
    \item RF-01: Registrar creación de objetos (CU-1)
    \item RF-02: Registrar modificaciones de geometría (CU-2)
    \item RF-03: Detectar cambios de modo de edición (CU-3)
    \item RF-04: Registrar eliminación de objetos (CU-4)
    \item RF-05: Visualizar métricas de modelado (CU-5)
    \item RF-06: Exportar datos a CSV/JSON (CU-6)
    \item RF-07: Filtrar métricas por objeto/evento/tiempo
    \item RF-08: Notificar errores en datos
    \item RF-09: Activar/desactivar captura desde la UI
    \item RF-10: Registrar marcas temporales precisas
\end{itemize}

\subsection{Requisitos no funcionales (RNF)}
\begin{itemize}
    \item RNF-01: Eficiencia en captura de eventos
    \item RNF-02: Soporte de múltiples objetos y cambios complejos
    \item RNF-03: Persistencia de datos ante cierres inesperados
    \item RNF-04: Interfaz clara e intuitiva
    \item RNF-05: Código mantenible y modular
\end{itemize}

\subsection{Requisitos de interfaz de usuario (RIU)}
\begin{itemize}
    \item RIU-01: Panel de control de captura
    \item RIU-02: Panel de métricas con gráficos y tablas
    \item RIU-03: Configuración de exportación de datos
    \item RIU-04: Alertas de errores
\end{itemize}

\subsection{Requisitos legales y éticos (RLE)}
\begin{itemize}
    \item RLE-01: Cumplimiento normativa protección de datos
    \item RLE-02: Consentimiento explícito del usuario
    \item RLE-03: Uso de software libre respetando licencias
\end{itemize}
