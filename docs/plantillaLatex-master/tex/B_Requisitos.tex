\section{Introducción}

El presente anexo comprende los criterios que el sistema tiene que cumplir para estar listo. El propósito es explicar lo que proporcionan cada adición y las restricciones al que debe adherirse el sistema para operar adecuadamente dentro de Blender.

El sistema está formado por dos addons complementarios:

\begin{itemize}
    \item Un addon de recopilación de datos, encargado de registrar automáticamente eventos producidos durante el modelado 3D.

    \item Un addon de análisis y visualización, encargado de procesar los datos registrados y generar métricas, gráficos y representaciones visuales.
\end{itemize}

Para organizar la especificación se describen primero los casos de uso principales. Después se incluyen los objetivos generales, los diagramas del sistema y el catálogo de requisitos funcionales, no funcionales, de interfaz y legales.

\section{Objetivos generales del sistema}

A partir de los casos de uso definidos, se establecen los siguientes objetivos generales:

\begin{itemize}
    \item Registrar automáticamente los eventos relevantes que ocurren durante una sesión de modelado 3D.

    \item Guardar la información capturada de forma estructurada para poder analizarla posteriormente.

    \item Permitir el análisis temporal, espacial y estructural de la actividad del usuario.

    \item Generar métricas que ayuden a entender cómo evoluciona el proceso de modelado.

    \item Mostrar los resultados mediante gráficos 2D y visualizaciones tridimensionales dentro de Blender.

    \item Mantener una interfaz sencilla para activar la captura, cargar datos y consultar resultados.

    \item Reducir el impacto sobre el rendimiento de Blender durante el uso del sistema.

    \item Respetar la privacidad del usuario mediante consentimiento previo para la recopilación de datos.
\end{itemize}

\section{Catálogo de requisitos}

En este apartado se presentan los requisitos definidos para el sistema desarrollado. 

Para organizar mejor la información y facilitar su comprensión, los requisitos se han dividido en cuatro categorías: funcionales, no funcionales, de interfaz y legales o éticos.

\subsection{Requisitos funcionales}

Los requisitos funcionales describen las acciones que el sistema debe poder realizar.

\begin{itemize}
    \item \textbf{RF-01.} El sistema debe registrar la creación de objetos dentro de la escena.

    \item \textbf{RF-02.} El sistema debe registrar modificaciones geométricas realizadas sobre los objetos.

    \item \textbf{RF-03.} El sistema debe detectar cambios entre modos de trabajo, como \textit{Object Mode} y \textit{Edit Mode}.

    \item \textbf{RF-04.} El sistema debe registrar la eliminación de objetos de la escena.

    \item \textbf{RF-05.} El sistema debe permitir visualizar métricas relacionadas con el proceso de modelado.

    \item \textbf{RF-06.} El sistema debe permitir exportar o reutilizar los datos registrados mediante archivos CSV.

    \item \textbf{RF-07.} El sistema debe permitir filtrar o seleccionar información según archivos, métricas u objetos analizados.

    \item \textbf{RF-08.} El sistema debe mostrar avisos o información de estado cuando se produzcan errores de captura o procesamiento.

    \item \textbf{RF-09.} El sistema debe permitir activar y detener la captura de datos desde la interfaz.

    \item \textbf{RF-10.} El sistema debe registrar marcas temporales asociadas a los eventos capturados.

    \item \textbf{RF-11.} El sistema debe generar visualizaciones tridimensionales a partir de los datos analizados.

    \item \textbf{RF-12.} El sistema debe integrar las representaciones visuales dentro del entorno de Blender.

    \item \textbf{RF-13.} El sistema debe generar gráficos estadísticos 2D mediante herramientas de visualización.
\end{itemize}

\subsection{Requisitos no funcionales}

Los requisitos no funcionales indican las condiciones de calidad que debe cumplir el sistema.

\begin{itemize}
    \item \textbf{RNF-01.} El sistema debe capturar eventos de forma eficiente, evitando almacenar información redundante.

    \item \textbf{RNF-02.} El sistema debe poder trabajar con escenas formadas por varios objetos.

    \item \textbf{RNF-03.} Los datos generados deben mantenerse disponibles para su análisis posterior.

    \item \textbf{RNF-04.} La interfaz debe ser clara y fácil de utilizar para usuarios con conocimientos básicos de Blender.

    \item \textbf{RNF-05.} El código debe organizarse de forma modular para facilitar su mantenimiento y ampliación.

    \item \textbf{RNF-06.} El sistema debe minimizar el impacto sobre el rendimiento de Blender durante las sesiones de modelado.

    \item \textbf{RNF-07.} El sistema debe permitir revisar manualmente los datos generados cuando sea necesario.
\end{itemize}

\subsection{Requisitos de interfaz de usuario}

Estos requisitos describen los elementos principales que debe ofrecer la interfaz del sistema.

\begin{itemize}
    \item \textbf{RIU-01.} El sistema debe incluir un panel de control para iniciar y detener la captura de datos.

    \item \textbf{RIU-02.} El sistema debe incluir un panel de análisis para cargar archivos CSV y consultar métricas.

    \item \textbf{RIU-03.} El sistema debe ofrecer opciones para exportar o seleccionar archivos de datos.

    \item \textbf{RIU-04.} El sistema debe mostrar información de estado, mensajes de error o avisos relevantes.

    \item \textbf{RIU-05.} El sistema debe proporcionar controles para generar visualizaciones 2D y 3D.

    \item \textbf{RIU-06.} El sistema debe incluir información de depuración que facilite comprobar el estado del logger durante el desarrollo.
\end{itemize}

\subsection{Requisitos legales y éticos}

Estos requisitos están relacionados con el uso responsable del sistema y la protección de los datos generados.

\begin{itemize}
    \item \textbf{RLE-01.} El sistema debe informar al usuario sobre la recopilación de datos antes de iniciar la captura.

    \item \textbf{RLE-02.} El sistema debe solicitar consentimiento explícito para registrar información durante la sesión de modelado.

    \item \textbf{RLE-03.} Los datos recopilados deben utilizarse únicamente con fines académicos, analíticos o de investigación.

    \item \textbf{RLE-04.} El desarrollo debe respetar las licencias de Blender, Python y las librerías externas utilizadas.
\end{itemize}
